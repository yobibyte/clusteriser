Кластеризатор группирует последовательности, входящие в выравнивание на группы, которые легче поддаются исследованию глазами и руками, чем огромные файлы.

==Краткая информация==

'''Вход:''' FASTA последовательность.

'''Результат:''' Папка (имя папки - текущий TIMESTAMP) с FASTA выравниваниями. Один файлик - один кластер. Название состоит из группы {fs|ws|we}_{номер кластера}. fs - full sized, ws - without start, we - without end.
 
==Мануал==

Программа запускается в командной строке со следующими параметрами:

* -h | --help - для вывода справки по программе
* -g | --gap - процент, последовательностей, которые будут отнесены в "без начала" и "без конца". Требует оба параметра
* -p | --percent - если параметр присутствует, то предыдущий ключ считается в процентах, и наоборот. По умолчанию используется гэп в символах.
* -c | --coeff - коэффициент похожести. Минимальное значение, необходимое, чтобы последовательности попали в одну группу.
* -f | --file - путь к входной fasta последовательности
* -r | --register - если ключ проставлен, то маленькие буквы считаются гэпом.
* -a | --alphabet <имя алфавита> - позволяет выбрать редуцированный алфавит из списка имеющихся:
    * SE_B_14
    * SE_B_10
    * SE_V_10
    * LI_A_10
    * LI_B_10
    * SOLIS_D_10
    * SOLIS_G_10
    * MURPHY_19
    * SE_B_8
    * SE_B_6
    * DAYHOFF_6
    * REGULAR (нередуцированный)
    * <абсолютный путь до файла> (перечисленные через запятую в одну строку группы, например: QWER,SD,F. Регистр не важен.). 
      Если происходит какая-то ошибка при считывании алфавита, то используется REGULAR.

Из вышеуказанных параметров только -f является обязательным. Все остальные при отсутствии проставляюется по умолчанию.

Значение параметров по умолчанию:
* -g = 50 50 
* -c = 0.4
* -r = по умолчанию регистр не учитывается
* -a = REGULAR

Пример использования программы:
java -jar Clusteriser-0.0.2-jar-with-dependencies.jar -g 10 10 -c 0.5 -f /home/yobibyte/Development/EclipseWorkspace/clusteriser/Misc/Data/testdata.fasta




По выполнении программы, в текущей директории создается папка result (если еще нет), в ней папка текущего времени, куда записываются файлы с результатами.

==Архитектура приложения==
* Старую кластеризацию по длине оставил (когда мы отсекаем последовательности по количеству гэпов), но сейчас в коде она нигде не вызывается.
* Для парсинга параметров командной строки используется [http://habrahabr.ru/post/123360/ Apache commons cli]
